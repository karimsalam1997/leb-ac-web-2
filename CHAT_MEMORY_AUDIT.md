## What The Memory Already Knows

The memory registry is useful and should be preserved. It already captures four rules that matter for this project:

- Archive tasks must produce real maps and manifests, not loose summaries.
- Transcript shelves must separate actual transcript bodies from metadata-only indexes.
- Script and voice work must rebuild sequence and rhythm before polishing sentences.
- Git and preview explanations must be plain English first, because the user is not a coder.

## Chat Classification System

Every useful chat should be classified into one of these buckets:

1. **Voice gold**
   - Raw phrases, angry insights, sentence rhythms, local images, and moments where the writing sounds like Karim before an assistant cleaned it.
   - Example targets: generator office, Sakiet el-Janzeer, Nahr el-Kalb, Fatima video, Solidere walk, Beirut Park design process.

2. **Essay seed**
   - A chat that contains a single strong essay spine.
   - It should become one essay packet, not seasoning for five essays.

3. **Source trail**
   - A chat where the value is links, documents, names, dates, pages, transcripts, or source-quality judgments.
   - This should feed a source note or archive manifest.

4. **Product decision**
   - Website, roadmap, editorial model, Signal Desk, image system, Sanity, or workflow decisions.
   - These belong in project docs, not prose.

5. **Bad AI artifact**
   - Smooth paragraphs, repeated binaries, fake-magazine tone, and overcomplete essays that should not be treated as Karim's voice.

6. **Duplicate or noise**
   - Repeated versions of the same export, old failed drafts, or conversations irrelevant to Lebanese Academic.

## Reusable Voice Lessons

The best local voice rules are already clear:

- Start inside the machine, not above it.
- One named Lebanese thing per paragraph when possible.
- Scene before thesis.
- Use "is" when "is" is the true verb.
- Concede the strongest opposing view in plain language, then cut through it.
- Avoid the assistant staircase: claim, clarification, caveat, implication, tidy close.
- Close on a thing: a balcony light, a census table, a blocked road, a shrine, a generator hum.

## Memory Cleanup Proposal

Add or strengthen future memory entries for:

- Canonical content source: `longform-essays.md` for essays, `launch-content.md` for letters/notebook.
- Build blocker: exclude `node_modules.offloaded-*` from TypeScript and lint.
- Signal Desk rule: dry run first, public refresh only when guard passes, always label stale public data.
- Archive rule: source bodies and metadata indexes are different.
- Voice rule: old chats are records of thinking, not a generic theme pantry.
- Product rule: do not pretend letters, notebook, or topics have full architecture until their routes exist.

Things future agents should ignore unless asked:

- Old dependency archives.
- `.next`, `tmp`, render-check images, and generated intermediate folders.
- Generic starter README instructions.
- Any essay draft that only restates the cartel/franchise frame without a new object.
